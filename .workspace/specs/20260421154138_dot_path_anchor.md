# Dot-path anchor for exec, envs, and get

> TLDR: Replace file-path anchors with dot-path anchors in exec/envs/get, removing the interactive picker and unifying the UX around content-based addressing.

**Status:** completed
**Created:** 2026-04-21
**Owner:** @oporpino

---

## Context

Currently `ward exec .ward/vault/staging.ward -- deploy` uses a file path as anchor. This couples the command to the physical file location rather than the logical content hierarchy. The real intent is "merge everything and scope to this dot-path" — which is content-based, not file-based. Also, the interactive picker in `ward get` was poorly received and should be removed in favour of shell TAB completion only.

## Objectives

- Replace file-path anchor with dot-path anchor in `exec`, `envs`, and `get`
- `ward exec qwert.environments.staging -- deploy` merges all vaults, extracts `qwert.environments.staging`, injects as env vars
- Shell TAB completion for dot-path argument in `exec`, `envs`, and `get`
- Remove interactive picker from `ward get`
- Add `--on-conflict=override` flag to `exec` and `envs` (default: error, same as today)
- `ward edit` and `ward new` keep file-path arguments unchanged

## Changes

### `internal/cmd/exec.go`
- Replace file-path anchor argument with dot-path string
- After full merge (no anchor), call `eng.GetAtPath(result, dotPath)` to scope
- Add `--on-conflict` flag (`error` default, `override` option)
- Add `ValidArgsFunction: completeDotPaths`

### `internal/cmd/envs.go`
- Same anchor change as exec
- Add `--on-conflict` flag
- Add `ValidArgsFunction: completeDotPaths`

### `internal/cmd/get.go`
- Remove `pickDotPath` interactive picker entirely
- Keep shell TAB completion via `completeDotPaths`
- Make dot-path argument required (or show all keys if omitted, no interactive)

### `internal/ward/engine.go`
- `Merge("")` already merges all vaults — no anchor filtering needed
- `GetAtPath(result, dotPath)` already scopes to a node — reuse as-is
- Add `--on-conflict` support: pass a `ConflictMode` to `Merge()` or handle at inject time

### `internal/config/config.go`
- Replace `Merge` field with `OnConflict`: `"error"` (default, omitted) | `"override"`
- Migrate legacy `merge: merge` → `on_conflict` omitted; `merge: override` → `on_conflict: override`
- CLI flag `--on-conflict` takes precedence over config value

### `internal/secrets/merge.go` (or conflict handling)
- Add `ConflictMode` type: `ConflictError` (default) | `ConflictOverride`
- When `override`: vault order in config determines winner (last vault wins)

## How to verify

```sh
# dot-path anchor — merges all vaults, scopes to path
ward exec qwert.environments.staging -- env | grep -i database

# TAB completion works
ward exec qwert.<TAB>   # shows available paths

# conflict override flag
ward exec qwert.environments.staging --on-conflict=override -- deploy

# get no longer shows interactive picker
ward get qwert.environments.staging   # prints value directly
ward get                               # prints all keys (no picker)

# edit and new unchanged
ward edit .ward/vault/staging.ward
ward new staging
```
