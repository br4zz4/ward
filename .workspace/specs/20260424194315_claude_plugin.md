# Claude Plugin for Ward

> TLDR: Add a `ward install claude-plugin` subcommand that fetches and installs the ward Claude Code plugin from `br4zz4/ai` into the user's project or home directory.

**Status:** completed
**Created:** 2026-04-24
**Owner:** @oporpino

---

## Context

Ward is not a well-known CLI, so Claude Code has no built-in knowledge of it. Devs using ward in their projects benefit from a plugin that gives Claude full context about ward's commands, vault structure, ancestry/merge rules, and common patterns.

The plugin lives in a centralized repo `br4zz4/ai` under `providers/claude/plugins/ward/` — so all br4zz4 Claude plugins are managed in one place. Other tools (infra, etc.) can add their own plugins there without scattering config across repos.

## Objectives

- Create repo `br4zz4/ai` with structure `providers/claude/plugins/ward/`
- Add `ward install claude-plugin` subcommand that fetches the plugin from `br4zz4/ai`
- Plugin installs a `CLAUDE.md` with complete ward context
- Plugin installs a `ward:debug` skill for inspecting conflicts, ancestry, and merge issues
- User chooses install location: project (`.claude/`) or global (`~/.claude/`)

## Changes

**`br4zz4/ai` repo (new):**
- `providers/claude/plugins/ward/CLAUDE.md` — ward context: commands, vault structure, dot-path syntax, ancestry rules, merge/conflict behavior, WARD_KEY, config reference
- `providers/claude/plugins/ward/skills/ward_debug.md` — skill to inspect a dot-path for ancestry chain, merge result, conflicts, and source file origins

**`br4zz4/ward` repo:**
- `internal/cmd/install.go` — new `install` command with `claude-plugin` subcommand; fetches files from `br4zz4/ai` via GitHub raw URLs
- `cmd/ward/main.go` — register `NewInstallCmd()`

**Install flow:**
1. `ward install claude-plugin` prompts: "Install for this project (.claude/) or globally (~/.claude/)? [project/global]"
2. Downloads `CLAUDE.md` and `skills/ward_debug.md` from `br4zz4/ai`
3. Writes to chosen location
4. Prints confirmation with next steps

## How to verify

1. Run `ward install claude-plugin` in `~/workspace/wehive/infrastructure/`
2. Choose project or global
3. Open Claude Code in that project
4. Ask Claude "how do I create a new ward vault for staging?" — Claude should answer correctly without extra context
5. Run `/ward:debug` and confirm it inspects ancestry correctly
